Streams

App for Tablet iOS and Android

Overview

Role: Discovery, UX, UI Design

Infor Ming.le Streams is an important core product for Infor that enables you to organize recurring processes with the ability to involve internal and external participants. As a designer on the Ming.le pod, my job was to design the native tablet experience for Streams for both iOS and Android. The tablet experience allows participants to organize processes on the go. Stream participants can collaborate seamlessly to complete activities, create new streams from templates, organize documents by threading them around each activity and quickly view the status of all tasks.

Challenge

The biggest challenge for designing the tablet version of Streams was not having a good user experience foundation to build from the desktop application. It was a bit outdated in its design and lacking in overall usability. The main points of frustration for our users was not being able to see all streams at once and quickly get the status of each stream. It was also difficult to sort through current streams based on the filter views being obscure icons.

From our initial research with current Streams users, we knew we needed to simplify the streams creation process for the tablet experience and this meant cutting features while making sure users could still complete all core Streams tasks on the go.

We knew after reviewing the desktop application we needed to simplify the streams creation process for the tablet experience....while making sure users could still complete all core Streams tasks on the go.

Solution

The end result was a native tablet experience that streamlined the following processes: allowing the user to see an overview of all active streams, create a new stream from a premade templates, and limited customization options for editing the steps in each stream. I sketched the initial screens to define these user flows. Later, I created prototypes of these user flows to use for user testing and eventually the handoff to our developer team. I was able to quickly iterate the design of both Android and iOS versions of this application based on user feedback.

Optimize for touch

Draggable steps allow the user to quickly edit and reorder their steps. This was essential for the Stream creator to be able to do while on the go.

Back to basics

Adding a progress bar to the dashboard page let users get a more thorough overview of each stream step which was added functionality.

Make it personal

The desktop design had no images or job titles for participants. I updated the design to include friendly faces as well as titles so you can see who you are working with.
