Ming.le Chat Application

Collaboration app for Ming.le Users

Overview

Role: Discovery, UX, UI Design

Infor Ming.le Chat was a major project that I worked on this year. Our goal was to replace the current Ming.le collaboration platform with a more modern chat platform that would integrate with our ERPs. We designed a seamless chat tool that will allow users to share data, files, and conversation with their teammates while improving their day-to-day workflow.

Challenge

This project required some extensive user research. The current Ming.le collaboration wasn't being adopted by a lot of our users despite being a free feature of the Infor Cloudsuite. We conducted surveys and user interviews to gather information about the current Ming.le and validate our hypothesis that adding a chat feature would make this application more usable.

Solution

The data we collected found that most users were paying for other third-party applications like Skype or Slack to communicate with their teams. We knew that by adding these capabilities to Infor's cloudsuite we could convert more users. The finished product was a chat application that include topic-based group coversation and a full team directory for 1-on-1 chats. The development of this appliation is still in process.

Blank state view

It's important to design for not only ideal states but Blank states as well. This was a view of the Blank state of the teammate panel when you haven't started a conversation yet.

Chat on the go

The responsive version of this app was important so users could chat on the go. We are still working on the responsive components of our styleguide so this was one of the first responsive applications that we were able to test on users.

Updated Notifications menu

Before the notifications for Ming.le were all mixed on one screen. We separate the Ming.le chat notifications so they could stand on their own and users could access from anywhere in Infor.

Fun Reactions

Because of some restrictions of certain industries, we had to work around full emoji support. Instead we offered standard reactions that users could choose to use to react to messages.
