Infor Responsive Navigation

Scheduling web application

Overview

Role: Discovery, UX, UI Design

Infor Ming.le is the centralized space for collaboration, business process improvement, and contextual analytics. Fully embedded within ERP, financials, and other organizational systems, this platform enables single-sign on for all Infor CloudSuiteâ„¢ solutions. My task was to create a responsive web experience for the Ming.le shell that encompasses the entire Infor experience. This include the app switcher, share functionality, search and user settings and bookmarks.

Challenge

My challenge was to create a seamless experience for our users across all devices. All of our products are already responsive or in the process of being designed for responsive, so the wrapper that holds them all needed to match. Things I focused on were designing for scale - whether a user had 100 applications or only 1 and optimize core functinalitlies for touch on smaller devices.

The responsive navigation went through many drafts and prototypes before landing on ideal navigation

Solution

I started this design process by working with our Information Architect to define the most important functions of the shell. His reserach determined the placement of each feature in the responsive navigaiton. We moved the app switcher, user settings and bookmarks to the main mobile nav with a bento box button and kept the search and share funtionality in the top header for instant access.

While designing the main mobile and tablet navigation, I was inspired by the Curtain navigation I had read about from Playstation.com. We needed to fit a lot of information into a small menu and the traditional accordian menu wasn't going to work. I wanted users to be able to scroll through a potential long list of applications while still be able to access other menu options. After some internal user testing around the office, this curtain model ended up being the perfect solution.

Create a familiar mobile experience

Even though this wasn't a native app we wanted to give the users a familiar experience. This slide up menu for connecting users is just one example.

Full screen search

The full screen search and share mimic the destop modals but create a seamless experience.

Curtain menu

This curtain menu was ideal for our use-case. One primary set of links, each containing a group of secondary links. Both levels of links are displayed at the same time.
